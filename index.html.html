How this works
The dashboard calls your Kinetic REST API through Automation Studio or directly via the REST endpoint. It reads Jobs, Resources, Shop Load, and Production Calendars, runs the AI recommendation engine client-side, and displays results. Actions (priority changes, resource reassignment, holds) push back to Kinetic through the same API.

Scheduling Actions
▶
Schedule Forward
Forward schedule from the Start Date. The engine places scheduling blocks into the earliest available capacity slots moving forward.
POST ScheduleEngineSvc → Direction: Start
◀
Schedule Backward
Backward schedule from the Due Date. Scheduling blocks placed so the last operation finishes by the required date.
POST ScheduleEngineSvc → Direction: End
✕
Remove from Schedule
Clears all scheduling blocks from resources. Job stays released but unscheduled. Frees capacity immediately.
PATCH JobEntry → JobScheduled: false
